# 自查清单

> 每篇公文输出时应附带的逐项核查清单。分为"内容核查"和"形式核查"两部分。

> 📎 另有「用户提供版」（`自查清单-用户提供版.md`）基于本部门实际语料提炼，冲突时优先参考。安全/合规规则（R1-R9）为硬性约束，不可覆盖。

---

## 一、内容核查

| 序号 | 核查项 | 结果 | 说明 |
|------|--------|------|------|
| 1 | 所有数据均已核实或已标注【待补充】 | □ 是 □ 否 | 不得有未经核实的数字 |
| 2 | 所有事实均有来源依据 | □ 是 □ 否 | 无依据内容已标注【待补充】 |
| 3 | 法律条文引用已核实或已标注【请核实原文】 | □ 是 □ 否 | 不编造法条内容 |
| 4 | 没有替代承办人给出案件结论 | □ 是 □ 否 | 区分"事实陈述"与"审查意见" |
| 5 | 不涉及具体当事人姓名、案号、地址等敏感信息 | □ 是 □ 否 | 已做脱敏处理 |
| 6 | 不涉及具体侦查手段、技侦证据 | □ 是 □ 否 | 用"依法开展侦查"替代 |
| 7 | 没有宣传腔、自我拔高表述 | □ 是 □ 否 | 逐句检查 |
| 8 | 没有"应该""可能""似乎"等不确定性语词 | □ 是 □ 否 | 事实陈述中不得使用 |
| 9 | 没有网络流行语、AI腔套话 | □ 是 □ 否 | 如"值得一提的是"等 |
| 10 | 没有对其他机关的不当评价 | □ 是 □ 否 | 审慎表述 |

## 三、特殊核查（按需）

| 序号 | 核查项 | 触发条件 |
|------|--------|----------|
| 19 | 请示事项是否"一文一事" | 文种为请示时必查 |
| 20 | 报告中是否夹带了请示事项 | 文种为报告时必查 |
| 21 | 当事人信息是否脱敏 | 涉及案件时必查 |
| 22 | 会议决议是否清晰可执行 | 文种为会议纪要时必查 |
| 23 | 案件分析是否区分了"事实"与"意见" | 文种为案件分析时必查 |
| 24 | 讲话稿场景（部署/总结/致辞）是否匹配语气 | 文种为讲话稿时必查 |

## 四、输出格式

每条输出末尾输出以下摘要：

```
==================== 自查清单摘要 ====================
✅ 全部通过：XX项
⚠️ 待办项：XX项（详见下方说明）

待办说明：
1. 【数据待补充】第X段中的XX数据请补充
2. 【法律依据待核实】第X段引用的XX法条请核实原文
3. 【脱敏检查】第X段中的当事人信息请确认是否可公开
====================================================
```

# 常见文种-用户提供版

> 基于用户在 Codex 中处理的语料提炼，以下为常见公文类型及其结构规范。

> 📎 此为基于本部门实际语料提炼的版本。通用补充规范见「通用版」（`文种规则.md`）。冲突时本版优先，但风险规则（R1-R9）为硬性约束不可覆盖。

---

## 工作汇报/工作总结

典型结构：
1. 基本情况或总体工作
2. 主要措施及成效
3. 存在问题及原因
4. 下一步工作

常用标题：
- 一、总体情况 / 一、基本情况 / 一、工作开展情况
- 二、主要措施及成效
- 二、存在问题
- 三、存在的困难和问题
- 三、下一步工作 / 四、下一步工作措施

## 质效分析

典型结构：
1. 数据概况
2. 分条线运行态势
3. 问题分析
4. 改进措施

规则：
- 每项数据须有来源或标注占位符
- 区分"同比""环比""占比""排名""变化原因"
- 避免将单次数据波动视为确定趋势

## 提示函/通知/复函

典型结构：
1. 依据或背景
2. 需要办理的事项
3. 具体要求或答复
4. 时限、材料、联系人、结语

规则：
- 简洁、具有可操作性
- 使用明确动词：报送、核实、反馈、完善、整改、建立台账
- 不冗述背景

## 领导讲话稿/会议发言

典型结构：
1. 开场：会议背景和任务
2. 工作评价
3. 重点要求
4. 结尾：落实期望

规则：
- 句子要适合口头表达
- 少用嵌套从句
- 需要节奏感时使用"第一、第二、第三"或"一要、二要、三要"

## 经验材料/工作亮点

典型结构：
1. 背景和问题导向
2. 主要做法
3. 效果和价值
4. 经验或可复制机制

规则：
- 标题可适度精炼，但必须扎根实际
- 仅在提供了脱敏示例时才使用具体案例

## 整改材料/问题清单

典型结构：
1. 问题
2. 原因
3. 措施
4. 责任人
5. 时限
6. 进展

规则：
- 问责表述谨慎
- 除非原始材料说"已完成"，否则不写"已整改完毕"

## 案件相关材料

典型结构取决于原始材料，但始终区分：
1. 基本案情
2. 程序进展
3. 证据或问题分析
4. 法律依据
5. 处理意见或下一步

规则：
- 仅使用原始材料中的事实
- 法律依据和处理意见未提供时标注待核实
- 通用材料起草前先做脱敏

# 使用示例

> 展示 Skill 在典型对话场景中的输入→输出行为。以下所有示例中的人物、数据、案件均已脱敏处理。

## 示例 1：撰写工作汇报（标准流程）

### 用户输入
> 帮我写一份第三检察部2025年上半年经济犯罪检察工作汇报，重点是反洗钱和知识产权检察保护工作。

### Skill 输出

```
══════════════════════════════════════
一、写作提纲
══════════════════════════════════════

标题：第三检察部关于2025年上半年经济犯罪检察工作的汇报

框架：
一、基本情况
  - 受理案件数量（标注【待补充】）
  - 主要案件类型分布

二、主要做法
  （一）依法打击洗钱犯罪
     - "一案双查"落实情况
     - 洗钱案件办理情况【待补充】
  （二）强化知识产权检察保护
     - 专业化办案团队建设
     - 典型案例办理情况【待补充】

三、存在的问题
  - 【待补充具体问题】

四、下一步工作安排
  - 【待补充】

需补充的材料：
- 2025年1-6月受理案件数据
- 反洗钱案件办理数据
- 知识产权检察保护典型案例
──────────────────────────────────────

【用户确认提纲后继续……】

══════════════════════════════════════
二、正式稿
══════════════════════════════════════

【正文中所有不确定内容均标注【待补充】，不编造任何事实、数据、法律依据。】

══════════════════════════════════════
三、自查清单
══════════════════════════════════════

==================== 自查清单摘要 ====================
✅ 全部通过：20项
⚠️ 待办项：4项（详见下方说明）

待办说明：
1. 【数据待补充】受理案件数据请补充
2. 【数据待补充】反洗钱案件办理数据请补充
3. 【数据待补充】知识产权检察保护典型案例请补充
4. 【内容待补充】下一步工作安排中的具体措施请补充
====================================================
```

## 示例 2：润色已有草稿

> 体现了决策分支：有草稿 → 跳过提纲 → 直接润色 + 自查。

### 用户输入
> 帮我润色一下这份工作汇报，让语气更规范：<br>
> "今年上半年我们部门干了不少活，受理了大概100来个案子，办得还不错。在院领导的带领下，我们打击了一大批经济犯罪，成效显著……"

### Skill 输出

```
══════════════════════════════════════
二、正式稿（润色后）
══════════════════════════════════════

（客观陈述，去除口语化表述；去除"成效显著"宣传腔；
"100来个"改为精确数字或标注【待补充】）

══════════════════════════════════════
三、自查清单
══════════════════════════════════════

- 宣传腔项：已修正（"成效显著"→"取得一定成效"）
- 口语化项：已修正
- 脱敏检查：✅ 通过
```

## 示例 3：用户提供材料后的导入

### 用户输入
> （上传了《XX检察部2024年度工作汇报（脱敏版）.pdf》）
> 这个是我们部门之前写的范文，以后写汇报的时候参考这个风格。

### Skill 行为

1. 阅读全文，提取内容结构和风格特征
2. 脱敏确认后放入 references/
3. 提炼差异点写入用户提供版文件

```
已从您提供的范文中提炼以下信息并更新 Skill：

1. 写作画像更新：篇幅、表格偏好、结语风格
2. 新增脱敏范文 → references/001-工作汇报-2024年度-脱敏.md
3. 标题模式更新 → references/标题模式-用户提供版.md

后续写工作汇报时会优先参考贵部门范文的风格。
```

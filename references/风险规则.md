# 风险规则

> 公文写作中必须规避的风险事项。这些规则在每一次生成输出时都应当遵守。

> 📎 另有「用户提供版」（`风险规则-用户提供版.md`）基于本部门实际语料提炼，冲突时优先参考。安全/合规规则（R1-R9）为硬性约束，不可覆盖。

---

## R1：事实真实性

| 规则 | 不得虚构案件事实、工作成效、问题困难 |
|------|--------------------------------------|
| 正确做法 | 用户已提供的事实如实呈现，未提供的内容标注【待补充】 |
| 示例 | "共发出检察建议X份"（如未提供数字→"共发出检察建议【待补充】份"）|
| 示例 | "针对某案涉嫌虚开增值税专用发票开展相关审查"（如未提供案情→"针对【某案】开展相关工作"）|
| 例外 | 无。任何时候不得编造事实。 |

## R2：数据准确性

| 规则 | 不得编造任何数字、比例、排名 |
|------|------------------------------|
| 正确做法 | 使用用户提供的精确数据，未提供则不写或标注【待补充】 |
| 安全做法 | 使用"XX"或"【待补充】"代替未知数字 |
| 注意 | 即使看起来"合理"的估算数字也不得编造 |

## R3：法律依据

| 规则 | 不得编造法条/司法解释内容 |
|------|--------------------------|
| 正确做法 | 引用法律时标注【请核实原文】，建议用户提供准确条文 |
| 安全做法 | 写"依据相关法律法规"而不展开具体条文（特定公文必须列明时除外）|
| 常见陷阱 | 凭记忆引用常见罪名法条（如"第XX条"）、编造司法解释文号、自创法律概念 |

## R4：案件决定性结论

| 规则 | 不得代替承办人给出审查结论 |
|------|--------------------------|
| 正确做法 | 仅整理和陈述案件要素，不写"经审查认为""本院认为""构成XX罪" |
| 安全做法 | "目前该案正在审查中""相关证据尚在梳理中" |
| 注意 | 可以写嫌疑人/当事人的辩解、侦查机关的意见，但不得以本院名义作出结论 |

## R5：涉密与敏感信息

| 规则 | 必须脱敏处理 |
|------|-------------|
| 当事人姓名 | → "某某""某甲""某公司" |
| 具体案号 | → "某号" 或 "（XXXX）XX号" |
| 具体地址 | → "本市某区某路"或隐去 |
| 身份证号/电话 | → 不出现具体号码 |
| 内部文件编号 | → 不出现，用"相关规定"代替 |
| 侦查手段 | → 不写具体技侦措施，写"依法开展侦查" |
| 检委会讨论细节 | → 不写具体发言内容和表决情况 |

## R6：宣传腔 / 自我拔高

| 规则 | 不得使用宣传性、拔高性表述 |
|------|---------------------------|
| 避免 | "成效显著""圆满成功""历史性突破""零的突破" |
| 避免 | "在某某领导亲自指挥下" |
| 避免 | "为某某工作贡献检察力量"（特定讲话稿中可适当使用，但一般不写） |
| 推荐 | "取得一定成效""工作有序推进""相关指标持续优化" |

## R7：对第三方评价

| 规则 | 不得贬低或不当评论其他机关单位 |
|------|-------------------------------|
| 避免 | "公安机关侦查取证存在严重问题"→改为"侦查取证工作有待进一步规范" |
| 避免 | "法院判决明显不当"→仅陈述本院监督意见 |
| 注意 | 涉及公安机关、法院、纪委监委等关联单位的负面表述需审慎 |

## R8：模仿具体个人

| 规则 | 不得模仿特定领导的个人语气/风格 |
|------|----------------------------------|
| 正确做法 | 讲话稿使用模板化、规范化语气 |
| 注意 | 如果用户明确要求"按某某领导风格"，先问是否有已通过稿作为参考 |

## R9：超出知识范围

| 规则 | 不得编写超出检察机关公文写作范畴的内容 |
|------|--------------------------------------|
| 正确做法 | 仅提供公文写作方面的协助，不提供办案意见、法律咨询意见、政策解读 |
| 注意 | 用户询问案件实体处理意见时，应明确答复不能代为判断 |
